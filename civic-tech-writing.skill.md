# 公民科技寫作技能

## 概述

此技能協助完成兩類文章的完整寫作流程：
1. **國際公民科技案例研究**：研究並介紹世界各地的 civic tech、govtech、參與式民主、開放資料等案例
2. **工作心得文**：記錄與反思工作經驗、學習歷程、專案觀察

寫作風格：**文體流暢易讀、結構清晰、資訊準確**，適合部落格或 Medium 等平台發表。

---

## 寫作流程

### 階段一：主題規劃

協助使用者釐清文章核心：
- 這篇文章想傳達什麼核心觀點或問題意識？
- 目標讀者是誰（一般大眾、公務員、科技從業者、研究者）？
- 預計文章長度與深度（概覽型 vs. 深度分析型）？
- 是否有特定案例或事件想聚焦？

輸出：確認主題方向與寫作目標

---

### 階段二：素材查詢

根據主題進行資料蒐集，涵蓋：

**國際案例文章**：
- 搜尋目標國家/城市的官方資料、學術研究、媒體報導
- 查找 mySociety、Code for All、OECD、GovTech Surge 等組織的相關報告
- 尋找第一手資料：官方網站、GitHub 儲存庫、評估報告

**工作心得文**：
- 整理使用者提供的背景脈絡與事件
- 查找相關研究或他人類似經驗作為對照

素材整理格式：
```
【素材清單】
- 來源：[標題 / 機構]
- 連結：[URL]
- 重點摘要：[2-3 句話]
- 用途：[預計用於文章哪個部分]
```

---

### 階段三：整合個人心得

在素材與使用者自身觀點之間建立連結：
- 詢問使用者對素材的看法與補充
- 協助提煉「這對台灣/自己工作有什麼啟示？」
- 確保文章有個人視角，而非純粹資訊彙整

---

### 階段四：大綱規劃

根據素材與心得，建立清晰的文章架構：

```
【文章大綱】
標題：（提供 2-3 個候選）
副標題：（選用）

一、引言（問題意識 / 吸引讀者的開場）
二、[主體段落 1]
三、[主體段落 2]
四、[主體段落 3]（視需要增減）
五、結語（啟示 / 行動呼籲 / 未來展望）

預計字數：XXXX 字
```

確認大綱後再進入草稿階段。

---

### 階段五：建立草稿

此階段以**結構穩健、資訊正確**為優先：
- 忠實呈現素材，標註資訊來源
- 確保論點有依據，避免過度推論
- 段落之間邏輯連貫
- 暫不追求文字優美，先確保內容完整

草稿完成後列出：
- 待確認的資訊或來源
- 需要使用者補充的個人經驗段落
- 結構上的疑慮

---

### 階段六：精修

此階段以**易讀性與文字通順**為核心：

**UX Writing 原則（易讀性優先）：**
- 以讀者為中心，用讀者熟悉的語言，避免術語堆疊
- 每個段落只傳達一個核心概念，避免資訊過載
- 重要資訊放前面，細節放後面（倒三角結構）
- 標題與段落首句要能獨立傳達意義，方便掃讀
- 善用換行與留白，讓視覺上有呼吸感

**語言規範（台灣繁體中文）：**
- 全程使用台灣用詞，嚴格避免中國慣用詞彙，常見對照：
  - 軟件 → 軟體｜視頻 → 影片｜網絡 → 網路
  - 鏈接 → 連結｜點擊 → 點選｜信息 → 資訊
  - 獲取 → 取得｜平臺 → 平台｜賦能 → 培力
- 如不確定某詞是否為台灣用法，以台灣媒體日常用語或教育部國語辭典為準

**讓文字更有人味：**
- 書名號「」只在必要時使用（如直接引述、特定專有名詞），不過度強調
- 避免詞彙堆積，如「創新、突破、前瞻、卓越」這類形容詞連用
- 語氣要像在跟讀者說話，而不是在交報告
- 適時加入轉折語氣，讓行文更自然（如「不過」、「說起來」、「其實」）

精修後提供：
- 完整修訂版全文
- 簡短說明主要修改方向

---

### 階段七：翻譯（繁體中文 → 英文）

此階段將完成的中文文章翻譯成英文，適合投稿國際平台或跨語言分享。

**翻譯原則：**
- 保留中文全部語意，不省略任何論點或細節
- 以意譯為主，不逐字直譯；可用更有效率的英文句型結構讓篇幅更精簡有力
- 專有名詞（如 g0v、vTaiwan）保留原名，第一次出現時補充簡短說明

**英文寫作規範：**
- 避免破折號（——或 --），改用句點、分號或重組句子結構
- 採用 UX Writing 原則：簡短句、主動語態、重要資訊優先
- 段落首句清楚點出該段主旨，方便掃讀
- 語氣保持專業但易讀，像在跟國際讀者說話，不像學術論文

**常見破折號替代方式：**
- 「A——B」→ 拆成兩句，或用 "A. B" / "A; B"
- 解釋補充 → 改用括號 (  ) 或獨立子句
- 強調轉折 → 改用 "However," / "In fact," / "That said,"

翻譯完成後提供：
- 完整英文版全文
- 若有翻譯上的詮釋選擇，簡短說明理由

---

## 寫作風格指南

- **語氣**：理性但有溫度，像一位有研究習慣的實務工作者在分享觀察
- **詞彙**：避免過度學術腔，也避免過度口語
- **結構**：每段有中心句，段落長度適中（4-8 行）
- **引用**：優先引用一手資料，並在行文中自然帶出來源
- **在地化**：國際案例要連結台灣脈絡，讓讀者感受到相關性

---

## 使用方式

使用者可以從任何階段切入，例如：
- 「我有一個模糊的主題想寫」→ 從階段一開始
- 「我有素材了，幫我整理大綱」→ 從階段四開始
- 「草稿寫好了，幫我潤稿」→ 直接進入階段六
- 「幫我查 XX 國的公民科技案例」→ 執行階段二
- 「幫我把這篇文章翻譯成英文」→ 直接進入階段七

每個階段完成後，詢問使用者是否確認繼續下一步。
